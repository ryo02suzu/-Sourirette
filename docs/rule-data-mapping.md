# 公式ルールデータ → ルールテーブル 対応表

最終更新: 2026-06-14

`src/billing/rule-tables.ts` は算定ルールの「解釈器（器）」で、ルールはデータ（テーブル）として
投入する設計。本書は、取得する公式データ（取得用プロンプトの①〜⑦）を、どのテーブルに
流し込むかの対応を示す。データが届いたら、この対応に沿ってローダーを書けば機械的に載る。

| 公式データ（出典） | 何を定義するか | 流し込み先（RuleTables） | 備考 |
| --- | --- | --- | --- |
| 歯科診療行為マスタ 関連別紙（背反テーブル） | 併算定不可の組合せ | `mutualExclusions[]` | scope（同日/同月）は別紙の区分から |
| 同（包括テーブル） | 包括される行為 | `inclusions[]` | includingCode が includedCode を包括 |
| 同（回数・きざみ関連） | 算定上限回数 | `frequencyLimits[]` | per（日/月）と maxCount |
| 電子点数表（歯科）の通則 | 算定回数・算定可否 | `frequencyLimits[]` / `mutualExclusions[]` | 通則の機械可読化分 |
| 施設基準（告示・通知＋別表5） | 加算の届出前提 | `facilityGates[]` ＋ `clinic.ts` のカタログ | requiredStandard＝届出コード |
| 支払基金 コンピュータチェック対象事例 CSV | 公式の機械可読チェック | 各テーブルへ振り分け | 本部点検条件を解釈して分類 |
| 審査情報提供事例（病名↔処置） | 病名適応（白/黒リスト） | `diagnosisRequirements[]` | 既定 warning（審査裁量があるため） |
| 記載要領 別表I（摘要欄必須コメント） | コメント必須コード | （別途）コメント必須テーブル | 返戻#13対策。器の追加候補 |

## 投入時の原則

1. **点数はマスタを正とする**（テーブルには点数を持たせない。点数は `master` から引く）。
2. **病名整合・査定リスク系は warning**（審査委員裁量があり、誤検知でブロックしない）。
3. **形式・回数・施設基準・背反は error**（客観的に判定でき、提出前に直すべきもの）。
4. **各テーブル項目は歯科医師レビューを経て有効化**（自動取込→レビュー→有効、の2段）。
5. **改定追従は valid_from 世代管理**（旧テーブルを閉じ、新テーブルを追加）。
