# 病名トークン curation バックログ（黙って捨てないための可視化）

`loaded.unresolvedDiseaseTokens`（`reportUnresolvedDiseaseTokens`）が、算定ルール調査DBの
病名トークンのうち**コードに解決できなかったもの**を露出する。解決できないトークンを含む
不適応ルールは**警告が発火しない**（=その分だけ効いていない）。誤検知を避けるため
fuzzy（部分一致）解決はしない方針なので、ここに挙がったものは**個別に正しいコード集合へ
curation（できれば歯科医師確認）**して有効化する。

## 現状の未解決トークン（2026-06-14 時点・19件）

歯科の略号・カテゴリ名（「破折」「脱離」等）は、傷病名マスタでは複数の具体コードに分かれる
（例: 破折 → 歯冠破折5218009 / 外傷性歯の破折8844150 …）。単一コードに一意化できないため
未解決。curation では「そのカテゴリに属するコード群」を明示して解決する。

## やり方（取込 → 検証 → 有効化）

1. 各 token を傷病名マスタ名称で検索し、該当する**コード群**を確定する（カテゴリは複数コード可）。
2. `disease_master` に略号エイリアスとして追加するか、`diagnosis_procedure` の token を
   `"略号/コード"` 形式へ具体化する。
3. 歯科医師レビュー後、`npm test` の `unresolvedDiseaseTokens` 件数が減ることを確認。
