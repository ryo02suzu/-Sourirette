# 歯科レセコン一体型電子カルテ 機能カタログ

既存製品（Dentis / With（メディア）/ POWER5G / WiseStaff / Opt.one / fine-SEED /
iQalte / Hi Dental Spirit / HAPPY ACTIS 等）の公開情報を調査して整理した機能一覧。
UI 設計（Claude design への入力）と開発スコープ判断の元ネタにする。
最終更新: 2026-06

凡例:
- ★ = MVP（Phase 1–2 で必要。これがないと「電カル＋レセコン」と呼べない）
- ◇ = 標準機能（競合はほぼ全部持っている。Phase 3–4）
- ○ = あると強い（差別化・後回し可）
- 🤖 = AI で楽にできるポイント（Sourirette の差別化軸）

---

## 1. 受付・予約

- ◇ 予約台帳（チェア・担当者別のタイムテーブル、週/日表示）
- ◇ Web 予約（患者が自分で取る）・予約変更・キャンセル
- ◇ 来院リマインド（SMS / LINE / メール）、無断キャンセル管理
- ◇ Web 問診（事前入力 → カルテに自動取込）🤖 問診内容から主訴・病名候補を要約
- ◇ 受付（来院チェックイン、当日の流れ：受付 → 診療中 → 会計待ち のステータス管理）
- ○ CTI（電話着信で患者情報ポップアップ）
- ○ キャンセル待ち・リコール枠の自動提案 🤖

## 2. 患者管理・保険

- ★ 患者基本情報（氏名・生年月日・連絡先・カルテ番号）
- ★ 保険情報の期間管理（主保険・公費併用、月途中の保険変更）
- ◇ オンライン資格確認（マイナ保険証。資格情報の自動取込・枝番対応）
- ◇ 保険証 OCR / 自動取込（手入力の排除）
- ★ 家族・世帯管理、アレルギー・全身疾患・服薬・禁忌の管理
- ○ 患者メモ・申し送り（受付⇔診療室の連絡）

## 3. カルテ記録（電カル本体）

- ★ SOAP 入力（テンプレート、定型文、前回コピー）
- ★ 🤖 **音声 → AI 下書き生成（dentia 連携。ここが最大の差別化）**
- ★ 処置入力（部位＋処置のセット入力、よく使うセットの登録）
- ★ 傷病名管理（開始日・転帰・部位、修飾語）
- ★ 確定処理（歯科医師の承認で final 化・追記専用・訂正履歴）
- ◇ 手書きメモ・シェーマ描画（タブレット＋ペン。With は140種以上のシェーマ）
- ◇ 画像添付（口腔内写真・レントゲンをカルテに貼る）
- ○ 問題志向型カルテ（POMR）、治療計画の立案・進捗管理
- ○ 🤖 カルテ記載の充足チェック（指導・監査で必要な記載が揃っているか）

## 4. 歯科固有チャート

- ★ 歯式チャート（口腔内図をタップして部位選択、現在歯・欠損・補綴物の状態表示）
- ★ 処置履歴の歯ごとの時系列表示（この歯に何をしてきたか）
- ◇ 歯周検査（P検）：ポケット1点/4点/6点法、出血・排膿・動揺度・プラークスコア、経時比較グラフ
- ○ 矯正・小児（乳歯列/混合歯列）表示切替

## 5. 算定・入力チェック（レセコンの心臓部）

- ★ 処置入力からの自動点数算定（点数マスタ参照、加算・包括・逓減）
- ★ 部位・病名・処置のトライアングルチェック（「この病名でこの処置はあり得ない」を防止）
- ★ 算定回数チェック（月1回・日1回等の上限、同月初診チェック）
- ◇ 施設基準による算定分岐（届出の有無で算定可否・点数が変わる）
- ◇ リアルタイムチェック（入力時）＋会計前チェック＋月次一括チェックの3段構え
- ◇ 自動病名候補（処置から対応病名を提案）🤖 AIでカルテ本文からも提案
- ◇ 数量・範囲の妥当性評価（義歯等で部位を指定単位でカウントし指定範囲内かを判定）
- ◇ 摘要欄（特記事項）の自動生成 — 90種超の摘要項目をカルテ記載から組み立てる
  🤖 生成AIでカルテ本文→摘要文の下書きを生成（返戻対策の本丸）
- ◇ 処方監査（既往歴・アレルギーと薬剤の禁忌・相互作用チェック）
- ○ 🤖 算定漏れの提案（「この処置をしたなら○○も算定できます」）
- ○ 🤖 過去データからの入力ナビ（この医師が・この所見で・どのコードを選ぶかの学習）

## 6. レセプト（月次請求）

- ★ レセプト作成（患者×保険単位、月締め処理）
- ★ レセプトチェック（エラー＝提出ブロック / 警告＝要確認、エラーからカルテへ直接ジャンプ）
- ★ 電子レセプト出力（UKE ファイル、PDF プレビュー）
- ◇ オンライン請求（支払基金・国保連への送信）
- ◇ 返戻・再請求処理（オンライン返戻再請求、月遅れ請求、赤伝処理）
- ◇ 公費・併用レセプト、高額療養費
- ○ 🤖 返戻理由の解析と修正提案

## 7. 会計・窓口

- ★ 窓口負担金の自動計算（負担割合・公費・領収書/診療明細書の発行 ※発行は義務）
- ◇ 自費診療・物販の管理（自費メニュー、見積書）
- ◇ 日計表・月計表・未収金管理
- ○ キャッシュレス決済・自動精算機・POSレジ連携
- ○ 矯正等の分割払い計画管理

## 8. 文書発行

- ★ 歯科疾患管理計画書（管理料算定に必須の文書）
- ◇ 診療情報提供書（紹介状）🤖 カルテから自動下書き
- ◇ 薬剤情報提供文書、処方箋
- ◇ 各種同意書・治療計画書・術前術後説明文書 🤖 患者向けの平易な説明文を自動生成

## 9. 訪問診療・介護保険

- ◇ 訪問先でのモバイル入力（タブレット・スマホ）＋ オフライン入力→自動同期

## AI 差別化の本線（Sourirette = 既存機能 × AI）

既存メーカーの機能は上記でほぼ網羅されており、機能の量では勝負しない。
**「入力・チェック・文書」の3つの面倒を AI で消す**のが本線：

1. **入力を消す**: 録音 → 文字起こし → SOAP 下書き → 処置・病名候補の自動抽出（dentia 統合）
2. **チェックを賢くする**: ルールベースの算定チェック（基盤実装済み）＋ AI による算定漏れ・記載不備の指摘
3. **文書を自動化する**: 管理計画書・紹介状・患者説明文をカルテから自動下書き

AI はすべて「下書き・提案」に留め、確定は人間（SaMD 非該当の維持。master-plan.md §2.5）。

## UI 画面マップ（Claude design に渡す画面単位）

日常業務の動線は「受付 → 診療 → 会計 → 月次」。画面は大きく7つ：

1. **ホーム / 当日ボード**: 本日の予約・来院ステータス（受付済/診療中/会計待ち）一覧
2. **患者サマリ**: 基本情報・保険・アレルギー/全身疾患・歯式・直近の経過が1画面に
3. **診療画面（カルテ入力）**: 左に歯式チャート＋傷病名、中央に SOAP/処置入力、
   右に AI パネル（録音ボタン → 下書き → 処置・病名候補 → 算定プレビューと警告）
4. **歯周検査画面**: P検の専用入力（数値の高速入力 UI、前回比較）
5. **会計画面**: 算定結果 → 負担金 → 領収書/明細書発行
6. **レセプト画面（月次）**: チェック結果一覧（エラー/警告）→ 該当カルテへジャンプ → UKE 出力
7. **設定/管理**: 職員・権限・施設基準の届出・マスタ更新状況・監査ログ

デザインの方向性メモ:
- 診察中はチェアサイドで触る → タブレット前提のタップ優先 UI、片手操作
- 受付・会計・レセプトは PC 前提の密度の高い一覧 UI
- 「AI が出したもの」と「人が確定したもの」を視覚的に明確に分ける（下書き=点線/淡色、確定=実線）
- エラー（算定不可）と警告（要確認）の2色ルールを全画面で統一
