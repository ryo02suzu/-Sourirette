# 商用化レディネス判定（Sourirette）

最終更新: 2026-06-14

このドキュメントは「いつ商用化できるか」に正直に答えるための判定表です。
**結論を先に: 現時点では商用化できません。** コードで詰められる部分は相当進みましたが、
商用化を阻む関門の大半は「コード外（規制・契約・臨床・運用）」にあり、エンジニアリング
だけでは越えられません。以下、関門ごとに「済 / 要コード / コード外」で正直に区分します。

凡例: ✅ 済 ／ 🟡 コードで対応可（未完） ／ 🔴 コードでは解決できない（外部要件）

## 商用化までのクリティカルパス（コードで動かせない順）

1. **オンライン請求の実接続と確認試験合格** — 閉域網・証明書・支払基金/国保連との手続き。
2. **公式マスタの正式ライセンスと改定運用** — 点数を「サンプル値」から外す前提。
3. **3省2ガイドライン／電子保存3原則の第三者担保** — 監査・認証。
4. **実医院でのβ運用で返戻ゼロ月** — 臨床・実務の検証。
5. **算定ルールの量産と臨床レビュー** — 数百〜数千ルール、誤算定の責任管理。

このうち #1〜#4 はコードでは完了させられません（外部の環境・契約・検証・認証が要る）。
私（エンジニアリング）が押し切れるのは #5 のルール拡充と、各機能の実装精度向上までです。

## 正直な現在地

- **レセ電出力のコア**は、一次仕様に忠実・テスト付き・自己点検付きで、技術的には実用水準に
  近づいています（確認試験未通過なので「実用」とは言い切れません）。
- ただし**「商用化できる」状態ではありません**。上記 🔴 が未達で、これらは設計・実装の巧拙
  ではなく、規制対応・契約・実医院検証・認証という別レイヤーの作業です。
- したがって「商用化できたら教えて」への誠実な回答は: **まだです。** コード側で残るのは
  ルール拡充と返戻再請求の項目解釈程度で、商用化の可否を決めるのは上の 🔴 群です。
