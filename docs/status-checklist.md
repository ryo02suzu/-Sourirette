## 次の一手（コードで進められる順）

機械可読データ（電子点数表3表・別表Ⅰ・主要マスタ）の取込は一巡。残りは性質が変わる:
1. 病名↔処置の適応（審査情報提供事例）→ prose解釈。高確度の少数を警告ルール化（誤検知注意）
2. 施設基準ごとの加算（告示・通知）→ prose解釈。データ化＋歯科医師レビュー
3. アプリの点数を実マスタ由来へ（サンプル値の全廃）／傷病名コード検証をHS生成に統合
4. 公式エンジン工場をアプリ/サーバへ接続

※ ここからは prose 解釈＋歯科医師レビューが中核で、速度は落ち慎重さが要る。

※ いずれも **取込 → 歯科医師レビュー → 有効化** の順を守る（間違いを医院に配らない）。
