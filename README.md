# Sourirette — レセコン一体型 歯科電子カルテ（フルスクラッチ）

**Sourirette** は、カルテ記載から診療報酬の自動算定・レセプト電算出力までを一体で扱う、
歯科専用のクラウド型電子カルテを **一からオリジナルで** 作るプロジェクトです。

- 姉妹プロジェクト: [dentia](https://github.com/ryo02suzu/dentia)（録音 → AI SOAP 下書き生成）。
  Sourirette はその「下書き」を受け取り、**正式なカルテ記録と保険請求**に変える本体です。

> ⚠️ 現在は **設計＋基盤コードの骨格（Phase 0）** の段階です。実際の保険請求に使える状態ではありません。

## UI（デモアプリ）

`app/` に診療画面・当日ボード・会計・レセプトの動く UI があります（Vite + React）。
飾りのモックではなく、**コアの算定エンジン（`src/billing`）が実際にバンドルされ、
処置の追加・傷病名の有無に応じてリアルタイムに点数・エラー・警告を計算**します。

```bash
cd app
npm install
npm run dev      # → http://localhost:5173
```

| 画面 | 内容 |
| --- | --- |
| 当日ボード | 予約→受付→診療中→会計待ちの流れ、オン資確認状況 |
| 予約 | アポイントブック（チェア×時間グリッド、処置種別の色分け） |
| 患者 | 検索一覧・患者サマリ（申し送り付箋・来院履歴・リコール） |
| 診療 | 歯式（オドントグラム）/ 傷病名 / SOAP / ✦AIアシスタント / 算定プレビュー＋処方監査（禁忌チェック） |
| 歯周検査 | P検 4点法フルマウス入力・🎙音声入力デモ・コアエンジンによる集計/病態評価（`src/domain/perio.ts`） |
| 会計 | 窓口負担計算・領収書/診療明細書プレビュー（標準様式の費用区分） |
| レセプト | 月次点検（エラー=提出ブロック / 警告=要確認）・✦摘要欄AI文案 → **算定エンジンから記録条件仕様準拠の UKE ファイルを実生成・ダウンロード**（Shift_JIS） |
| 経営分析 | 売上推移・✦キャンセル予測・リコール対象＋✦AI文案生成 |

追加画面: **訪問診療**（医療×介護の同時算定シミュレータ・ケアマネ報告書✦AI下書き）/ **文書発行**（管理計画書・紹介状等の✦AI下書き→交付記録）/ **技工**（クラウド指示書・技工士チャット→診療録記録）。
UX: 画面状態の保持・トースト（アンドゥ付き）・⌘Kコマンドパレット・P検→診療連動・歯面単位入力（M/D/B/L/O）・施設基準トグル→算定エンジン即時反映・📞CTI着信デモ。

スクリーンショットは [`shots/`](./shots/) を参照。再生成は `cd app && npm i --no-save playwright && node shot.mjs`。
AI は現段階ではモック実装（`app/src/services/ai.ts`）。本番は dentia と同じ
2段構成（文字起こし + Claude による下書き生成）をサーバ経由で呼び出します。
点数はすべて ⚠サンプル値（公式マスタ取込は Phase 2）。

## リポジトリ構成

```

src/receipt/         レセプト電算（UKE）: レコード定義・Shift_JIS・組立・自己点検・返戻再請求

```

## 開発

ランタイム依存はゼロ（Node 22 標準機能のみ）。開発依存は TypeScript と型定義だけです。

```bash
npm install     # devDependencies（typescript / @types/node）のみ
npm run build   # tsc でコンパイル
npm test        # ビルド + node --test（96 テスト）
```

## 設計上の絶対原則

1. **点数・コードをコードにハードコードしない。** 算定ルールは支払基金の公式マスタ
   （診療行為・傷病名・医薬品マスタ）を参照する。改定はマスタ差し替え＋ルール差分で追従する。
2. **確定したカルテ記録は変更も削除もしない（追記専用）。** 訂正は新しい版の追加で表現し、
   ハッシュチェーンと監査ログで真正性を担保する（電子保存の3原則）。
3. **AI は下書きまで。** 診断・算定の確定は必ず歯科医師の確認操作を経る。
